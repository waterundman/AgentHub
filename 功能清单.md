# AgentHub 功能清单 (v1.0.0 - v1.3.0)

## v1.0.0 功能

### 1. LLM Router
- **文件**: `src/services/llmRouter.js`
- **功能**: 统一的 LLM API 网关层，支持多 Provider 配置、路由策略和故障转移
- **关键API**: 
  - `LLMRouter` 类：核心路由类
  - `route(providerName, modelId, options)`: 路由请求到指定 Provider
  - `routeWithFallback(model, options)`: 使用路由策略自动选择 Provider
  - `callAgentWithRouter()` / `streamCallAgentWithRouter()`: 兼容现有接口的包装函数
- **验证点**: 
  - 支持 13 个 Provider 配置
  - 支持 4 种路由策略（成本优先、延迟优先、故障转移、手动选择）
  - 支持流式和非流式请求
  - 支持超时和重试机制

### 2. 断路器 (Circuit Breaker)
- **文件**: `src/services/circuitBreaker.js`
- **功能**: 实现 CLOSED/OPEN/HALF_OPEN 三状态断路器，用于 LLM Provider 故障检测和自动恢复
- **关键API**:
  - `CircuitBreaker` 类：单个断路器实例
  - `CircuitBreakerManager` 类：管理多个 Provider 的断路器
  - `call(fn)`: 执行受断路器保护的请求
  - `getState()`: 获取当前状态
  - `getStats()`: 获取断路器状态摘要
- **验证点**:
  - 支持状态自动转换（CLOSED → OPEN → HALF_OPEN → CLOSED）
  - 支持可配置的失败阈值、恢复超时、成功阈值
  - 支持断路器管理器统一管理多个 Provider
  - 提供默认配置（13 个 Provider）

### 3. 路由策略
- **文件**: `src/services/routingStrategy.js`
- **功能**: 实现 4 种路由策略：成本优先、延迟优先、故障转移、手动选择
- **关键API**:
  - `RoutingStrategy` 枚举：COST, LATENCY, FALLBACK, MANUAL
  - `CostRoutingStrategy`: 成本优先策略
  - `LatencyRoutingStrategy`: 延迟优先策略
  - `FallbackRoutingStrategy`: 故障转移策略
  - `ManualRoutingStrategy`: 手动选择策略
  - `RoutingStrategyFactory`: 策略工厂
  - `LatencyTracker`: 延迟追踪器
- **验证点**:
  - 支持按成本排序选择最低成本 Provider
  - 支持按延迟排序选择最快响应 Provider
  - 支持按优先级顺序尝试 Provider
  - 支持用户直接指定 Provider 和模型
  - 支持延迟追踪和统计

### 4. 工具注册表
- **文件**: `src/services/toolRegistry.js`
- **功能**: 管理工具定义、权限和调用日志
- **关键API**:
  - `ToolRegistry` 类：工具注册表
  - `register(tool)`: 注册工具
  - `get(name)`: 获取工具定义
  - `list()`: 列出所有工具
  - `listByCategory(category)`: 按分类列出工具
  - `logCall(record)`: 记录调用日志
  - `getCallHistory(agentHash, limit)`: 获取调用历史
  - `getToolStats(agentHash)`: 获取工具调用统计
- **验证点**:
  - 预定义 15 个内置工具（文件读写、命令执行、HTTP 请求等）
  - 支持工具分类（read, write, execute, network, system）
  - 支持工具权限管理
  - 支持调用日志记录和统计

### 5. 工具调用拦截器
- **文件**: `src/services/toolCallInterceptor.js`
- **功能**: 验证工具调用权限，记录日志，强制调用限制
- **关键API**:
  - `ToolCallInterceptor` 类：工具调用拦截器
  - `validate(toolName)`: 验证工具调用是否允许
  - `isToolAllowed(toolName)`: 检查工具是否被允许
  - `isCallLimitReached(toolName)`: 检查是否达到调用限制
  - `wrapToolCall(toolName, toolFn)`: 包装工具调用，添加验证和超时
  - `withTimeout(fn, timeout)`: 创建超时包装器
- **验证点**:
  - 支持允许/禁止工具列表
  - 支持总调用次数和单工具调用次数限制
  - 支持调用超时控制
  - 支持调用统计和剩余次数查询

### 6. 前端图标系统
- **文件**: `src/components/Icon.jsx`
- **功能**: 集成 Lucide React 图标库，提供统一的图标组件
- **关键API**:
  - `Icon` 组件：通用图标组件
  - 导出常用图标：Play, Stop, Settings, GitBranch 等
- **验证点**:
  - 支持动态图标名称
  - 支持自定义大小、颜色、样式
  - 提供常用图标的直接导出

### 7. 工具使用标签
- **文件**: `src/components/ToolUseTag.jsx`
- **功能**: 显示工具调用结果的标签组件
- **关键API**:
  - `ToolUseTag` 组件：单个工具标签
  - `ToolUseTagList` 组件：工具标签列表
- **验证点**:
  - 显示工具名称、图标、耗时
  - 支持成功/失败状态显示
  - 支持点击事件

### 8. 工具权限沙箱
- **文件**: `src/components/AgentToolSandbox.jsx`
- **功能**: 配置 Agent 的工具权限和调用限制
- **关键API**:
  - `AgentToolSandbox` 组件：工具权限配置界面
  - 支持允许/禁止工具切换
  - 支持调用限制配置
- **验证点**:
  - 按分类显示所有工具
  - 支持单个工具的允许/禁止操作
  - 支持最大调用次数、单工具最大调用次数、超时配置
  - 显示当前配置摘要

### 9. Agent LLM 配置
- **文件**: `src/components/AgentLLMConfig.jsx`
- **功能**: 为单个 Agent 配置 LLM Provider 和模型
- **关键API**:
  - `AgentLLMConfig` 组件：Agent LLM 配置界面
  - 支持 Provider 和模型选择
  - 支持最大 Tokens 和 Temperature 配置
- **验证点**:
  - 支持全局配置和 Agent 级配置覆盖
  - 支持 Provider 切换时自动重置模型
  - 支持恢复默认配置

### 10. 转写规则引擎
- **文件**: `src/services/transcriptionRules.js`
- **功能**: 定义多种外部 Agent 项目格式的导入规则
- **关键API**:
  - `TRANSCRIPTION_RULES` 数组：预定义转写规则
  - `getRuleByFormat(format)`: 根据格式获取规则
  - `getRules()`: 获取所有规则
  - `getSupportedFormats()`: 获取支持的格式列表
- **验证点**:
  - 支持 10 种外部格式（Claude Code, OpenCode, Cursor, Windsurf, Markdown, AutoGPT, LangChain, CrewAI, AutoGen, Generic JSON）
  - 支持文件匹配和内容匹配
  - 支持数据转换器

### 11. 转写引擎
- **文件**: `src/services/transcriptionEngine.js`
- **功能**: 负责格式检测、规则匹配、数据转换和项目生成
- **关键API**:
  - `TranscriptionEngine` 类：转写引擎
  - `detectFormat(files)`: 检测文件格式
  - `transcribe(files, format)`: 执行转写
  - `extract(files, rule)`: 提取数据
  - `transform(data, rule)`: 转换数据
  - `generateProject(data, format)`: 生成项目结构
- **验证点**:
  - 支持自动格式检测
  - 支持多种文件格式解析（JSON, YAML, Python, Markdown）
  - 支持 10 种格式的转换逻辑
  - 生成标准的 AgentHub 项目结构

### 12. 导入向导
- **文件**: `src/components/ImportWizard.jsx`
- **功能**: 引导用户完成外部项目导入的向导组件
- **关键API**:
  - `ImportWizard` 组件：导入向导界面
  - 支持文件拖拽和选择
  - 支持格式检测和预览
  - 支持进度显示
- **验证点**:
  - 5 步导入流程（选择文件 → 格式检测 → 预览转写 → 导入中 → 完成）
  - 支持文件拖拽
  - 支持格式手动选择
  - 支持导入进度显示
  - 支持错误处理
